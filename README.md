4. How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?

- The first thing I think of when approaching this is organization. I would first separate resources from new documents (interview transcript separated from notes about interview transcript). From there, I would review and identify clear stated needs + additional inferred needs. For example, you say you want people to be able to create an account that has a username and password, but we infer that most systems need an email as a username, and a password that fits modern requirements such as length, numbers, special characters. I would definitely use these strategies and adopt methods of communication with the all parties involved to confirm everyone is on the same page.
